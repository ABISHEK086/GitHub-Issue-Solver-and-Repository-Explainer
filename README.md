# GitHub Repository Explainer

Paste any public GitHub repo and get back a plain-English summary, tech stack,
key components, entry points, and an architecture diagram (Mermaid.js) —
generated by a **free** Groq API key.

## How it works

1. Backend hits the **GitHub REST API** (free, no key required for public
   repos — a token just raises your rate limit) to pull the file tree, the
   README, and any manifest files it finds (`package.json`,
   `requirements.txt`, `Dockerfile`, etc.).
2. That context gets packed into a prompt and sent to **Groq's** free,
   OpenAI-compatible chat completions API running Llama 3.3 70B.
3. The model returns structured JSON, which the frontend renders — including
   a live Mermaid diagram of the architecture.

## 1. Get your free API keys

- **Groq (required):** https://console.groq.com/keys — sign up, create a key.
  Free tier is generous and fast (this project defaults to
  `llama-3.3-70b-versatile`).
- **GitHub token (optional but recommended):** https://github.com/settings/tokens
  → "Generate new token (classic)" → no scopes needed for public repos. Without
  a token you get 60 requests/hour from GitHub; with one, 5,000/hour.

## 2. Backend setup

```bash
cd backend
python -m venv venv
source venv/bin/activate      # Windows: venv\Scripts\activate
pip install -r requirements.txt

cp .env.example .env
# then edit .env and paste in your GROQ_API_KEY (and GITHUB_TOKEN if you have one)

python main.py
```

The API will be running at `http://localhost:8000`. Check it with:

```bash
curl http://localhost:8000/api/health
```

## 3. Frontend setup (Next.js)

```bash
cd frontend
npm install

cp .env.local.example .env.local
# edit .env.local if your backend isn't at http://localhost:8000

npm run dev
```

Then open **http://localhost:3000** in your browser. Paste a repo (e.g.
`fastapi/fastapi` or a full `https://github.com/...` URL) and click **Draft it**.

The frontend never calls the backend cross-origin — `next.config.js` rewrites
any `/api/*` request to your `BACKEND_URL`, so the browser only ever talks to
`localhost:3000`. That means no CORS setup to worry about, and swapping
backend hosts (e.g. after deploying) is a one-line env var change.

## Extending this

- **Swap the LLM provider:** the Groq call is a single OpenAI-compatible
  `POST` in `call_groq()` in `main.py` — pointing it at OpenRouter or Gemini's
  OpenAI-compatible endpoint is a one-line change to `GROQ_API` / model name.
- **Cache results:** repo analyses don't change often — add a simple
  SQLite/Redis cache keyed on `owner/repo` + latest commit SHA to avoid
  re-analyzing the same repo.
- **Deploy free:** backend → Render/Railway free tier or Fly.io; frontend →
  Vercel's free tier (built for Next.js — just set `BACKEND_URL` as an
  environment variable on the Vercel project).
- **Auth-gate it:** add a rate limit per IP if you deploy this publicly, since
  your Groq key is shared server-side.

## Troubleshooting

| Problem | Fix |
|---|---|
| `403` from `/api/analyze` | GitHub rate limit hit — add a `GITHUB_TOKEN` to `.env` |
| `500: missing GROQ_API_KEY` | You haven't set `GROQ_API_KEY` in `backend/.env` |
| Diagram doesn't render | Occasionally the model returns slightly invalid Mermaid syntax — click "Draft it" again |
| `fetch failed` / 404 on `/api/analyze` | Backend isn't running, or `BACKEND_URL` in `frontend/.env.local` doesn't match where it's running — restart `npm run dev` after changing `.env.local` |
